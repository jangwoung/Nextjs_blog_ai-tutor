# 学習者ガイド — Next.js ブログ チュートリアルの進め方

このドキュメントは、**Next.js ブログ チュートリアル**を Cursor や Firestudio などの AI エージェントと一緒に進めるときの「やり方」をまとめたものです。

---

## 1. チュートリアルでやること

- **開発プラン**（`docs/development-plan.md`）に沿って、Phase 0 から Phase 5 まで順に進めます。
- 各フェーズで「**概念の説明 → 簡単な例 → ブログ用の課題**」が出ます。**課題は「目的」と「できたときに見える結果」だけ示され、手順の番号リストは出しません。** 自分で考えて実装し、詰まったらヒントや段階的なガイドを頼みます。
- エージェントは**通常、あなたの代わりにファイルを書きません**。**自分でコードを書いて進める**前提です。  
  「このフェーズをスキップして」「〇〇のフェーズに飛びたい」と言ったときだけ、必要なコードを出してもらえ、**自分で貼るか、エージェントに適用させるか**を選べます。

---

## 2. セッションの始め方

- チュートリアル用のエージェント（Cursor のルールで `docs/blog-tutor-rules.md` を読ませている想定）を開いたら、  
  **「ブログチュートリアルを始めたい」** や **「Next.js ブログの講義を続けたい」** などと伝えます。
- エージェントが **プロジェクトのファイルを読んで**「今どこまで完了しているか」を判定し、  
  - 何もしていなければ **Phase 0** から、  
  - 途中までしていれば **次のフェーズ** から  
  課題を出します。
- 初回だけ「**経験レベル（初心者 / 中級 / 上級）**」を聞かれることがあります。自分のレベルに近いものを答えると、説明の詳しさが変わります。

---

## 3. 課題の進め方

1. **出された「目的」と「期待する結果」を読む**  
   「何を作ればよいか」「完了したときに画面でどうなっていればよいか」を把握します。
2. **自分で実装する**  
   必要なファイルを編集・作成します。開発プラン（`docs/development-plan.md`）や `specs/001-blog-baseline/` の仕様も参照してよいです。
3. **できたと思ったら**  
   「〇〇までやりました」「できたと思います」と伝えます。エージェントが **プロジェクトを読んで** 正しくできているか確認し、OK なら次のフェーズに進みます。
4. **詰まったら**  
   「ヒントが欲しい」「もう少し段階的に教えて」と伝えます。**答えを直接書かせず、ヒントや質問で考えさせる**形で助けてもらえます。
5. **コードは貼らなくてよい**  
   「コードを貼って」と言われなくても、エージェントは **リポジトリのファイルを直接読んで** 確認します。必要なら「このファイルを見て」とファイル名を指定する程度で十分です。

---

## 4. よく使う言い回し

| したいこと | 例 |
| ---------- | -- |
| 進捗を確認したい | 「今どこ？」「進捗を見たい」「目次を出して」 |
| 今の課題を飛ばしたい | 「このフェーズをスキップして」「このステップは省略して」 |
| 別のフェーズに飛びたい | 「Phase 3 からやりたい」「管理者画面のところに飛びたい」 |
| ヒントが欲しい | 「ヒントをください」「どこを直せばいいか教えて」 |
| 段階的に教えてほしい | 「手順を細かく出して」「ステップバイステップで」 |
| 答えを確認したい | 「この実装で合っているか見て」「レビューして」 |

---

## 5. スキップ・ジャンプしたとき

- **スキップ**（今のフェーズを飛ばす）  
  - エージェントが「スキップすると、こういう状態になります。よろしいですか？」と確認します。  
  - OK なら、そのフェーズが完了した状態の **コード変更一覧**（ファイルパス + 内容）を出します。  
  - **「自分で適用する」** か **「エージェントに適用させる」** を選びます。  
  - 適用後、**期待する結果**（画面で何が見えるか）を説明されるので、ブラウザで確認します。
- **ジャンプ**（別のフェーズに飛ぶ）  
  - 「Phase N に飛びたい」と伝えると、**その間にあたるフェーズが自動で完了扱い**になることが伝えられます。  
  - 必要なら、飛ぶ先に合わせた **セットアップ用のコード** を出してもらい、上と同様に「自分で適用 / エージェントに適用」を選びます。  
  - セットアップが終わったら、**そのフェーズのレッスン** が始まります。

---

## 6. コンテキストの「置き場所」（.idx のイメージ）

- **進捗の判定**は、リポジトリ内の **`.idx/learning-index.md`** に書かれた「Phase ごとのチェック項目」を見て行われます。  
  - ここに「Phase N が完了しているか」を判定するための **ファイル名やコードの目安** が書いてあります。  
  - Firestudio の `.idx` のように「**プロジェクトの文脈と進捗の索引**」として使う、というイメージです。
- **チュートリアルのルール**は **`docs/blog-tutor-rules.md`** にあります。  
  - エージェントの「ルール」や「システムプロンプト」にこのファイルを読み込ませると、**ステップごとのチェックと学習サポート**が開発プランに沿って動くようになります。
- **学習者の進め方**は、この **`docs/learner-guide.md`** にまとまっています。

---

## 7. 環境について

- チュートリアルは **Next.js（App Router）+ TypeScript + Supabase + Tailwind** を前提にしています。
- デプロイは **Cloudflare** を想定しているため、**サーバー側のファイル書き込み（`fs.writeFile`）や SQLite は使いません**。データは Supabase、画像は Supabase Storage に統一します。
- 詳細は `docs/development-plan.md` と `specs/001-blog-baseline/` を参照してください。

---

## 8. まとめ

- **始め方**: 「ブログチュートリアルを始めたい」と伝える → エージェントが進捗を判定し、Phase 0 または続きの課題を出す。  
- **進め方**: 目的と期待する結果を見て自分で実装 → できたら「できた」と伝える → エージェントがファイルを読んで確認。  
- **困ったら**: 「ヒント」「段階的に」と頼む。必要なら「このフェーズをスキップ」「Phase N に飛びたい」も使える。  
- **コンテキスト**: 進捗は `.idx/learning-index.md`、ルールは `docs/blog-tutor-rules.md`、進め方は `docs/learner-guide.md`。
