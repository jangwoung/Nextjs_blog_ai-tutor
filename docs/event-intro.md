## 標準紹介文（ connpass / Doorkeeper 等の「イベント説明」用）

### こんなイベントです

Next.js 16（App Router）と Supabase を使って、**公開サイト**と**管理者画面**がそろったブログを、フェーズを追いながら一緒に作るハンズオンです。

- **公開側**: トップに画像1枚＋本文の記事一覧・詳細（Markdown 表示）
- **管理側**: ログインして記事の投稿・編集・削除、下書き／公開の切り替え

データと画像は **Supabase（DB + Storage）** に統一し、デプロイは **Cloudflare** を想定しています。  
「本番で動く」形を最初から意識した構成なので、そのまま実務やポートフォリオに活かしやすい内容です。

### 学べること・やること

- Next.js 16（App Router）のページ構成と Route Handlers
- Supabase を使ったデータ取得・保存と Storage への画像アップロード
- 管理者認証（簡易パスワード＋署名付き Cookie）と middleware での保護
- Markdown 表示と XSS 対策、環境変数の扱い（クライアント／サーバー分離）

開発は **Phase 0（環境準備）→ Phase 5（仕上げ）** まで段階的に進めます。  
各フェーズで「何を作るか」「どこまでできれば OK か」がはっきりしているので、初心者でも迷いにくい構成になっています。

### 対象者

- Next.js や React に少し触れたことがある方
- App Router や Supabase を実践で使ってみたい方
- ブログや CMS っぽいものを一通り作ってみたい方

### 前提環境

- Node.js 20+
- エディタ（VS Code / Cursor 等）
- （任意）Supabase アカウント（当日説明しながら作成も可）

### こんな方におすすめ

- チュートリアルを「写すだけ」ではなく、自分で考えながら進めたい方
- Cloudflare や Supabase を Next.js と組み合わせた構成を体験したい方
- 講義・勉強会の題材として「ブログ構築」を検討している方

## 長めの紹介（ブログ・LP用）

### タイトル案

- **Next.js 16 で作る、Supabase × Cloudflare ブログ ハンズオン**
- **「読める＋管理できる」ブログを Next.js で一から作る会**

### 本文

Next.js でブログを作るイベントでは、**「記事を読む画面」と「記事を書く・管理する画面」の両方**を、一つのアプリで実装します。

**なぜ Supabase と Cloudflare なのか**

本イベントでは、デプロイ先に **Cloudflare（Pages / Workers）** を想定しています。  
Cloudflare の実行環境では、サーバー側でファイルに書き込む（`fs.writeFile` など）ことができません。  
そのため、「管理画面から投稿した内容をずっと残す」には、**データベース**と**ストレージ**が必須です。

そこで **Supabase** に統一します。

- 記事データ → Supabase の DB（posts テーブル）
- 画像 → Supabase Storage にアップロードし、URL を DB に保存

この組み合わせにすると、Cloudflare にそのままデプロイしても、記事と画像がきちんと永続化されます。  
「ローカルでは動くけど本番では動かない」を避けつつ、モダンなスタックを体験できます。

**フェーズで区切った学習**

開発は **Phase 0 〜 Phase 5** の 6 段階で進めます。

- **Phase 0**: 環境準備（Next.js、Tailwind、Supabase、環境変数）
- **Phase 1**: 公開ブログ（一覧・詳細、Markdown 表示、XSS 対策）
- **Phase 2**: 管理者画面の基盤（ログイン、middleware で保護）
- **Phase 3**: 記事の投稿・編集・削除、画像アップロード
- **Phase 4**: 下書きと公開の切り替え
- **Phase 5**: バリデーション、セキュリティ確認、README

各フェーズの「目的」と「ここまでできれば OK」が明確なので、**自分で手を動かしつつ、どこまで進んだか** を把握しやすい構成です。  
イベント後も、リポジトリとドキュメントをたよりに続きから取り組めます。

**AI と一緒に進めるオプション**

用意しているリポジトリには、**学習プラン**と**フォルダ構成の説明**がトップページに表示されます。  
さらに、Cursor などの AI エージェント用の「チュートリアルルール」と**進捗のチェックリスト**（.idx / learning-index）を同梱してあります。  
希望する方は、**AI に「ブログチュートリアルを始めたい」と伝えて、ヒントや次の課題をもらいながら進める**使い方もできます。  
（イベント本体は、AI なしで進めても成立する内容です。）
